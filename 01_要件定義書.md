# 要件定義書 ― つづけるリハ

| 項目 | 内容 |
|---|---|
| プロダクト名 | つづけるリハ |
| 種別 | PWA |
| 版 | 1.0（アプリ版数 v13 に対応） |
| 作成日 | 2026-06-20 |

---

## 1. 背景・目的

リハビリや自主トレは「毎日・複数項目・回数やセット単位」で行うため、紙やメモでは「今日どこまでやったか」「続けられているか」が見えにくい。本アプリは、続けたい運動を自分で登録し、タップだけで達成を記録し、達成率と継続日数を可視化することで、**自主トレの継続を支援する**ことを目的とする。

入院中の自主トレ管理を出発点とし、退院後や一般的な習慣管理にも使えるよう、項目を自由に編集できる汎用ツールとして設計する。

## 2. 対象ユーザー・利用シーン

| 区分 | 内容 |
|---|---|
| 主な利用者 | リハビリ中の本人。自分のスマートフォンで毎日記録する。 |
| 利用環境 | iPhone（Safari）を主対象。ホーム画面に追加し、アプリのように単独画面で利用。 |
| 利用シーン | 食後・就寝前などにメニューを開き、終わった項目をタップ。タイマーが要る運動はその場で計測。 |
| 利用頻度 | 1日に複数回（食後ごと等）。1セッションは数十秒〜数分。 |

## 3. 用語定義

| 用語 | 意味 |
|---|---|
| メニュー | 登録された運動項目とセクションの集合（＝設定）。 |
| セクション | 項目をまとめる見出し（例：「毎食後」「セットで行う運動」）。 |
| 項目 | 1つの運動（例：「右足の足上げ運動」）。 |
| 頻度 | 項目の数え方。「朝・昼・夜」「朝・夜」「1日N回」「N回（秒）×Mセット」。 |
| 枠／セット | 達成チェックの最小単位。頻度に応じて朝昼夜の各枠、または各セット。 |
| ユニット数 | 1項目の枠／セットの合計。1日の満点はメニュー全項目のユニット数合計。 |
| スナップショット | 各日付に保存される「その日のメニュー構成」。過去日の達成率を固定するために用いる。 |

## 4. 機能要件

| ID | 機能 | 内容 |
|---|---|---|
| FR-01 | メニュー作成・編集 | セクション・項目を自分で追加／編集／削除できる。 |
| FR-02 | 頻度設定 | 朝昼夜／朝夜／1日N回／N回（秒）×Mセット を選べる。 |
| FR-03 | タイマー設定 | 頻度とは独立に、なし／単発（分秒）／セットごと（キープ＋休憩）を付与できる。 |
| FR-04 | 並べ替え | 項目はセクション内でドラッグ並べ替え、セクションは上下移動できる。 |
| FR-05 | 達成記録 | 枠／セットをタップして達成を記録・解除できる。 |
| FR-06 | 進捗可視化 | 当日の達成率（リング）、達成数／合計、項目ごとの完了表示。 |
| FR-07 | コンプリート演出 | 全ユニット達成で祝福演出（紙吹雪＋バイブ）。 |
| FR-08 | 継続日数 | 連続して全達成した日数（ストリーク）を表示。 |
| FR-09 | タイマー実行 | 単発カウントダウン、またはセットごと自動進行（キープ→休憩、バイブ通知）。 |
| FR-10 | カレンダー履歴 | 月表示で日別の達成度を色分け。日付タップで内訳表示。月の集計（全達成日数・取組日数・達成率）。 |
| FR-11 | 履歴の固定 | メニューの追加・削除・変更は当日以降のみ反映し、前日までの達成率は変えない。 |
| FR-12 | バックアップ／復元 | メニューと記録をJSONで書き出し・読み込みできる。 |
| FR-13 | 当日リセット | 当日の記録だけを消去できる。 |
| FR-14 | 自動更新 | 公開側の更新を検知して自動で最新化する。 |

## 5. 非機能要件

| 区分 | 要件 |
|---|---|
| 対応環境 | iOS Safari を主対象（モダンブラウザで動作）。縦向き・スマホ画面前提。 |
| オフライン | Service Worker により通信なしでも起動・利用できる。 |
| 性能 | 端末内処理のみ。タップ操作は即時反映。 |
| 保守性 | 役割ごとにファイル分割（各ファイル200行未満）。素のHTML/CSS/JSでビルド不要。 |
| プライバシー | データは端末内（localStorage）に限定。外部送信・アカウント登録なし。 |
| 可用性／更新 | バージョン番号による自動更新。アプリ再インストール不要。 |
| アクセシビリティ | 大きめのタップ領域、明確なコントラスト、片手操作前提のレイアウト。 |

## 6. 制約条件

- バックエンド・データベースサーバを持たない（クライアント完結）。
- ビルドツール・トランスパイラを使用しない（素のHTML/CSS/JavaScript）。
- データ保存は端末の localStorage に限定。容量・永続性はブラウザ仕様に従う。
- すべてのファイルを同一階層（フラット）に配置する（スマホからの更新を可能にするため）。

## 7. スコープ外（本版で扱わない）

- 複数端末間の同期、クラウド保存、ユーザーアカウント／認証。
- 他者との共有、SNS連携、通知（プッシュ）。
- 医療的な診断・指導機能（本アプリは記録支援であり医療機器ではない）。

## 8. 前提・リスクと対応

| リスク | 対応 |
|---|---|
| ホーム画面アプリ削除でデータ消失 | バックアップ書き出し機能を提供し、UIで注意喚起。 |
| ブラウザのストレージ消去 | 同上（復元で戻せる）。 |
| 端末ごとにデータが分かれる | 仕様として明記。必要時はバックアップで移行。 |
| 長時間タイマー中の画面消灯 | 既知の制約として明記（必要なら経過時刻方式へ拡張可能）。 |
